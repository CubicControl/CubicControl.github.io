Step 1 ‚Äì Prep Your PC & Network

First, we‚Äôll enable Wake-on-LAN, grab your IP/MAC, and forward the ports so your world is reachable from the outside.

  1. Turn on Wake on LAN in BIOS
    Restart your PC and press your BIOS key (often Del, F2, or F10 while booting). In the Power or Advanced, APM Configuration settings, enable Wake on LAN or Power On By PCI-E. Save changes and exit.
  2. Turn on Wake on LAN in Windows
    Press Win + R ‚Üí type ncpa.cpl ‚Üí Enter.
    Right-click your Ethernet adapter ‚Üí Properties ‚Üí Configure.
    • Go to Power Management and check all boxes.
    • Go to the Advanced tab and set Wake on Magic Packet = Enabled and Wake on Pattern Match = Enabled.
  3. Find your IP and MAC address
    Open Command Prompt (CMD) and run:
    • ipconfig ‚Üí copy your IPv4 address.
    • getmac /v /fo list ‚Üí copy the Physical Address for the active Ethernet adapter (the one whose Transport Name does not say ‚ÄúMedia disconnected‚Äù).
  4. Forward ports on your router
    Open your router admin (often http://192.168.1.1) and log in (password is usually printed on the router). Find Port Forwarding / IPv4 Port Mapping and add:
    • TCP ‚Üí local IP (your IPv4):
      • Public 443 ‚Üí local 443
      • Public 80 ‚Üí local 80
    • UDP ‚Üí local IP:
      • Public 9 ‚Üí local 9
  5. Create your hostname on FreeDNS
    Go to freedns.afraid.org ‚Üí Sign Up and create an account. After logging in:
    • Click Subdomains ‚Üí Add.
    • Type = A.
    • Choose a subdomain name and a domain from the list.
    • Set Destination to your public IP (search ‚Äúmy IP‚Äù on Google to find it).
    • Save, and keep this hostname handy (e.g. myserver.chickenkiller.com).
  6. Install Playit
    Download Playit for Windows from playit.gg and install it.
    Note the full path to playit.exe (for example: C:\Users\You\AppData\Local\Programs\Playit\playit.exe).
If something doesn‚Äôt work later (like waking the machine), double-check Wake-on-LAN settings first: BIOS ‚Üí Windows adapter ‚Üí router.

Step 2 ‚Äì Install CubicControl (Server + Launcher)

Now we‚Äôll install CubicControl on your host machine and connect it to your server folder and Playit.

  1. Download & place CubicControl
    Download CubicControl.exe and put it in a folder you choose, for example C:\CubicControl.
  2. Run as Administrator (first time)
    Right-click CubicControl.exe ‚Üí Run as administrator.
    This creates a Task Scheduler entry so CubicControl can auto-start after wake.
  3. Enter your hostname
    When the terminal asks for a hostname, enter the FreeDNS hostname you created (e.g. myserver.chickenkiller.com). Caddy will use this for HTTPS and the ClientSide site.
  4. Create your panel password
    Open http://localhost:38000 in your browser and create your CubicControl panel password. (You can reset this later if needed.)
  5. Set environment variables
    When prompted in the panel, set:
    • ADMIN_KEY
    • AUTH_KEY
    Write down AUTH_KEY ‚Äì you‚Äôll use it when deploying the ClientSide website.
  6. Point CubicControl to Playit
    When asked for the Playit path, browse to your playit.exe and confirm it.
  7. Create a Minecraft profile
    In the CubicControl panel:
    • Click New Profile and give it a name.
    • Folder: point to your existing Minecraft server folder (where your .jar and world live).
    • Run script: use a run.bat with something like:
      java -jar jarname.jar nogui
      (Replace jarname with your actual server jar.)
    • Keep auto-close and sleep-after-inactivity enabled, and adjust the timeout if you want.
    • Save the profile, then click Start to test it.
  8. Keep CubicControl running
    Leave CubicControl.exe running.
    You can access the panel locally via http://localhost:38000 or remotely via your hostname. Don‚Äôt stop Caddy if you want remote or ClientSide access.
When no players are online for your chosen timeout, CubicControl will stop the server and put the PC to sleep. It wakes up again using Wake-on-LAN when someone asks to start it.

Step 3 ‚Äì Deploy CubicControl ClientSide (Render Free Tier)

Next, we‚Äôll deploy the web dashboard that lets you (and your friends) wake and start the server from anywhere.

  1. Sign in to Render
    Go to render.com and sign up or log in.
  2. Create a new Web Service
    On your Render dashboard:
    • Click New ‚Üí Web Service.
    • Source: Public Git Repository.
    • Repository URL: https://github.com/CubicControl/CubicControl-ClientSide
    • Build Command: pip install -r requirements.txt
    • Start Command: python app.py
    • Plan: choose the Free tier.
  3. Add environment variables
    In your Render service, add these environment variables (you can paste them as a block and then edit): 
    AUTH_KEY=YOUR_AUTH_KEY_HERE
TARGET_MAC_ADDRESS=AA:BB:CC:DD:EE:FF
TARGET_IP_ADDRESS=myserver.chickenkiller.com
LOGIN_USERNAME=yourlogin
LOGIN_PASSWORD=yourpassword
SERVER_NAME=YourServerName
ALLOW_WAKE=true
ALLOW_START=true
ALLOW_STOP=false
ALLOW_RESTART=false
    • AUTH_KEY must match the AUTH_KEY you set in CubicControl.
    • TARGET_MAC_ADDRESS is the MAC you copied from getmac.
    • TARGET_IP_ADDRESS is your FreeDNS hostname (or your public IP).
    • LOGIN_USERNAME / LOGIN_PASSWORD are what you‚Äôll share with friends who can control the server.
    • ALLOW_* flags define what they can do. For example, you might allow wake and start, but disable stop/restart.
  4. Deploy and grab the URL
    Click Deploy and wait for the build to finish. Once it‚Äôs done, copy the Render URL shown at the top of the service page ‚Äì this is your ClientSide link.
On the Render free tier, your site may ‚Äúsleep‚Äù when idle and take a few seconds to wake up when someone first visits.
